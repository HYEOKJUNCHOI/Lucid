## 2026-04-10 퀘스트 시스템 회의 결과

### 핵심 전환: 루틴 → 퀘스트

**왜 바꿨나:**
- "루틴" = 양치질 느낌, 가볍고 의무적 → 짧게 하고 "공부했다"는 **도피처** 됨
- "퀘스트" = 도전/미션 느낌, 해냈다는 무게감
- 코딩 공부가 무거운데 루틴으로 가볍게 포장하면 시니어가 보면 비웃음

**단어 차이:**
| | 루틴 | 퀘스트 |
|--|------|--------|
| 느낌 | 해야 하니까 | 깨야 하니까 |
| 완료 | "했다" | "클리어했다" |
| 대화 | 안 함 | "너 오늘 퀘스트 깼어?" |
| 안 했을 때 | 죄책감 | 아쉬움 (내일 또) |

### 퀘스트 시스템 설계

**형태:**
- 매일 1회 (일일퀘스트)
- 30분 목표 — 이름값에 맞는 무게감
- 4개 미션 구성
- 부분 완료 허용 → 완료한 만큼만 XP 받고 종료 가능
- [퀘스트 수락] 버튼으로 시작 (RPG 감성)

**미션 구성 예시:**
```
오늘의 퀘스트 [약 30분]
☐ 미션 1: 챕터 클리어         +150 XP
☐ 미션 2: 문제지옥 N문제       +150 XP
☐ 미션 3: 약점 보강            +100 XP
☐ 미션 4: 자유학습 주제        +100 XP
전체 클리어 보너스:          +200 XP
최대: 700 XP
```

**챕터 → 문제지옥 → 자유학습** 을 돌려가면서 사용하게 만드는 구조.

**디아2 "하나만 더" 심리:**
- 미션 1은 가볍게 (진입 마찰 0)
- 미션 2는 다른 맛 (지루함 방지)
- 미션 3은 짧게 (연장 유도)
- 미션 4는 보상 크게 (완주 유도)

### 원두 채굴 보상 시스템

**컨셉: "공부 = 채굴"**
- "공부하러 가자" → **"채굴하러 가자"**
- 공부의 노동감을 채굴의 기대감으로 프레이밍
- 채굴소/원두/로스팅 등 용어 통일

**드랍 룰 (확정):**
```
첫 퀘스트 완료:
  → 🫘 확정 드랍 1개 (튜토리얼, 궁금증 유발)

이후 퀘스트 완료 (전체 클리어 시에만):
  → 🫘 23% 확률 드랍 1개

조건:
  - 일 1회 (일일퀘스트니까)
  - 전체 클리어 시에만 드랍 (부분 완료는 XP만)
  - 스킵한 날은 기회 손실

교환:
  - 5개 모으면 ☕ 커피 1잔
  - 기수(약 110일, 5개월, 주5일) 평균 5잔
  - 체감: 한 달에 1잔
  - 평균 주기: 약 22일
  - 운 좋으면 10일, 운 나쁘면 35일+
```

**왜 23%인가:** 110일 ÷ 25개 원두 = 4.4일에 1개 = 23%

**피티 시스템 / 수료 보너스 / 주간 리셋 등 추가 장치 없음.** 순수 랜덤.

**UX 원칙:**
- 처음엔 원두가 뭔지 설명 안 함 → 학생이 스스로 발견
- 지갑에 조용히 쌓임 → "이게 뭐지?" 궁금증
- 5개 모이면 첫 알림: "교환 가능해요"

### 온보딩

**첫 진입 질문:** "어떤 걸 배우고 싶나요?"
- "배우고 있나요?" (수동) ❌
- **"배우고 싶나요?" (능동)** ✅
- 이유: 학원 학생뿐 아니라 독학생도 포함하는 범용 방향

**선택 방식:**
- 단일 선택 **추천** ("한 가지에 집중이 가장 빠름")
- 멀티 선택 가능 (강제 아님)
- 프로필에서 언제든 변경

**배치고사 선택:**
- "처음부터 천천히" / **"배치고사 바로 보기"** 둘 다 제공
- 처음부터 선택 → 아이언 시작, 나중에 배치고사 도전 가능

**멀티 선택의 의미:**
- 예: JAVA 수강 중이지만 취업 목표가 리액트+JS+JAVA → 3개 체크
- 수업은 JAVA만 따라가므로 JAVA 티어만 자연 상승
- 나머지(리액트, JS)는 아이언으로 남음 → 곁눈질로 보이며 호기심/자발 학습 유도
- 대시보드: 체크한 과목들의 티어만 표시 (전체 평균 X)

### XP vs LP 구분 (정리)

| 모드 | XP (→ 커피·원두) | LP (실력 티어) |
|------|:---:|:---:|
| 챕터별 학습 | ✅ | ❌ (오픈북이라 측정 불가) |
| 퀘스트 | ✅ | ✅ (적게) |
| 문제지옥 | ✅ | ✅ (많이) |
| 자유학습 | ✅ | ❌ |

**퀘스트와 문제지옥은 LP 공유** — 학생이 Java 선택 → 퀘스트에서 쌓은 LP가 문제지옥 Java 티어에 반영됨. 둘이 같은 지갑.

**전체 LP 없음.** 과목별 LP만 존재. 온보딩에서 체크한 과목만 티어가 보임.

**XP는 어디서 쌓든 = 커피/원두로 교환** (성실성 보상)
**LP는 맨몸 실력 측정 모드에서만** (실력 증명)

### 시즌 시스템 (확정)

**2개월 시즌제**

```
기수 (5개월 / 약 110일)
├── 시즌 1 (2개월) → 리셋
├── 시즌 2 (2개월) → 리셋
└── 시즌 3 (1개월, 마무리) → 수료

각 시즌 종료 시:
  - LP 0으로 리셋
  - 티어 리셋 (전 시즌 -1단계 배치 or 재배치고사 선택)
  - "시즌 X 최종: 골드" 기록만 박제

기수 종료 (수료) 시:
  - 전체 데이터 폐기

**왜 2개월인가:**
- 1개월 = 너무 짧아서 쌓는 맛 없음
- 5개월 = 너무 길어서 중반에 동기 꺼짐
- 2개월 = 쌓는 맛 + 리셋 긴장감 동시 확보
- 기수당 약 2.5번 시즌 → 적당한 리듬

**왜 리셋하나:**
- 영구 최종 티어는 굳이 필요 없음 (어차피 기수 끝나면 폐기)
- 리셋이 있어야 "다시 쌓아가는 맛" 생김
- 새 시즌 = 새 동기

**3개월 마일스톤은 그대로:** 시즌과 별개로 강사 면담 기준 ("3개월째 골드 못 가면 공부 안 한 것").

### 루틴의 새 정체성 정리

**퀘스트가 공부 자체는 아님.** 공부의 **입구**임.
- 퀘스트 미션 하나 → "어? 이거 기억 안 나네"
- → 챕터별 학습 30분
- → 문제지옥 티어 도전
- 진짜 공부는 챕터/문제지옥에서. 퀘스트는 트리거.

하지만 **퀘스트 자체도 30분 정도의 무게감**은 가져야 함. 이름값을 해야 하니까.
