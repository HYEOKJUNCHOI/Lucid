### LP 시스템 (문제지옥 전용)

**XP vs LP:**
- XP: 웹 전체 레벨 (Lv.1~99), 성실도 척도
- LP: 문제지옥 전용, 과목별 티어 (아이언~다이아), 실력 척도
- 문제지옥에서는 XP + LP 중첩 획득

**LP 시스템 (롤 방식 — 티어마다 100LP, 단가가 달라짐):**

| 티어 | 승급 LP | 정답 | 오답 | 체감 |
|------|---------|------|------|------|
| 아이언 | 100 | +10 | -3 | 몸풀기 |
| 브론즈 | 100 | +10 | -5 | 기초 확인 |
| 실버 | 100 | +10 | -5 | 복습했으면 OK |
| 골드 | 100 | +8 | -5 | 수업+복습 = 도달 |
| 플래티넘 | 100 | +7 | -10 | 벽 |
| 다이아 | 100 | +5 | -10 | 강사도 긴장 |

**설계 의도:** 골드까지는 성실하면 도달, 플래티넘부터 진짜 실력, 다이아는 남 가르칠 수준

**티어별 의미:**
- 아이언: 이제 시작
- 브론즈: 기초 개념은 앎
- 실버: 혼자 코드 읽을 수 있음
- 골드: 실전 문제 풀 수 있음
- 플래티넘: 코딩테스트/자격증 수준
- 다이아: 남 가르칠 수 있는 수준

- LP는 티어 승급 시 0 리셋
- 0 밑으로 떨어지면 강등
- LP 게이지 실시간 표시 (게이지 차오르는 애니메이션)
- 시즌 리셋: 1개월 주기, 배치 재진행

### 문제지옥 흐름

```
문제지옥 → 과목 선택 (Java 등)
  → 첫 진입: 배치고사 10문제 → 티어 배치
  → 이후: 해당 티어 난이도 문제 무한
    → 맞추면 +LP +XP, 틀리면 -LP
    → LP 가득 → 승급, LP 바닥 → 강등
  → 시즌 리셋: 1개월 주기, 배치 재진행
```

### 문제 수준 차이

| | 루틴 | 문제지옥 |
|--|------|----------|
| 목적 | "이 코드 이해했니?" | "이해의 깊이 측정" |
| 수준 | 오픈북 (코드 보면서) | 기초 범위 내, 질문 깊이로 난이도 조절 |
| 스타일 | 가벼운 확인 문제 | 같은 개념을 얼마나 깊이 아는지 |
| 도구 | AI채팅, 메타포, 힌트 전부 | 사전만 |

**핵심 원칙:** 코드는 기초 범위를 벗어나지 않음. 티어가 올라갈수록 질문의 깊이만 올라감.
- 골드 = 수업+복습 도달 기준선 = 수료 기준
- 플래티넘~ = 강사 커버 범위 밖, 학생 개인 역량
- 질문 깊이 세부 설계는 다음 회의(바바라 오클리)에서 검증 예정

### 문제지옥 생성 방식
- 한 문제씩 실시간 생성 (앞 결과 보고 난이도 조절)
- 문제 수 제한 없음 (무한, 나가고 싶을 때 나감)
- 세션 시작 시 선택 없이 바로 첫 문제
- 퀴즈 로딩은 문제창 영역에서 표시 (채팅창 X)

### 가독성 개선
- 문제 텍스트: 순백 + 폰트 업
- 선택지: 밝은 화이트
- 번호(1,2,3,4): 액센트 색
- 카드 간 여백 넉넉하게
- 호버/포커스 시 카드 밝아짐
- 코드블록 배경색 분리

### 사용자 경험 원칙
- 초보자: 해석, 메타포, 힌트 등 코어 기능이 적극 도움
- 숙련자: 코드+퀴즈+채팅이 한 곳에 있어서 편하다는 가치
- 문제지옥: "여기가 현실이야" — 친절한 웹에서 배운 걸 시험당하는 곳

### 강사 관점 (두 축 교차)
- streak 높은데 티어 낮음 → 열심히 하는데 이해 못 함 → 면담
- streak 낮은데 티어 높음 → 머린 좋은데 안 함 → 동기부여
- 둘 다 높음 → 놔두면 됨
- 둘 다 낮음 → 위험 학생
