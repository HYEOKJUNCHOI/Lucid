## 2. 비유 추천 알고리즘 — Epsilon-Greedy
> 📌 **롤모델 차용:** YouTube, Netflix의 콘텐츠 추천 시스템
> 핵심 원리인 **탐색-활용 균형(Explore-Exploit Tradeoff)** 을 비유 추천에 적용

### 배경
> "좋아요 1개 받은 비유를 모든 학생에게 계속 보여주는 건 비효율적이다."
> YouTube도 좋아요 많은 영상만 보여주지 않는다.

### 개념: 탐색(Explore) vs 활용(Exploit)

| 전략 | 설명 |
|------|------|
| **활용 (Exploit)** | 검증된 좋은 비유를 보여줌 → 학습 효과 보장 |
| **탐색 (Explore)** | 새로운 GPT 비유를 던져봄 → 더 좋은 비유 발굴 가능성 |

### 적용 방식

```
파일 열기
  ↓
random(0~1) 생성
  ├─ 0.8 이하 + 저장된 비유 좋아요 ≥ 2  →  Firebase 비유 재사용 (활용)
  └─ 0.2 이상 or 검증 비유 없음          →  GPT 새 비유 생성 (탐색)
                                              └─ Firebase에 저장 후 투표 대기
```

### 코드 구조 (예정)
```js
const shouldExplore = Math.random() > 0.8;
const cached = await getMetaphorDoc(repoName, filePath);

if (!shouldExplore && cached?.likes >= 2) {
  // 활용: 검증된 비유 사용
  return cached;
} else {
  // 탐색: GPT로 새 비유 생성
  const newMetaphor = await generateMetaphor();
  await saveOrUpdateMetaphor(...);
  return newMetaphor;
}
```

### 기대 효과
- 좋은 비유는 자연스럽게 살아남고 나쁜 비유는 자연도태
- 학생이 쌓일수록 비유 품질이 자동으로 향상
- GPT 호출 비용 절감 (검증된 비유는 재사용)

## 3. 비유 품질 점수 — Wilson Score
> 📌 **롤모델 차용:** Reddit의 댓글/포스트 정렬 알고리즘
> 단순 좋아요 수 대신 **통계적 신뢰도** 기반으로 품질 평가

### 배경
> 좋아요 3/3 (100%) vs 좋아요 950/1000 (95%)
> 단순 비율로는 3/3이 더 높지만, 신뢰도는 950/1000이 훨씬 높다.

### Wilson Score 공식
```
score = (p̂ + z²/2n - z√(p̂(1-p̂)/n + z²/4n²)) / (1 + z²/n)

p̂ = 좋아요 / 전체 투표수
z  = 1.96 (95% 신뢰구간)
n  = 전체 투표수
```

### 활용처
- 관리자 페이지에서 "베스트 비유" 정렬 기준
- Reddit 댓글 정렬과 동일한 방식

## 4. 퀴즈 문제 수 — GPT 자율 결정 (예정)

### 현재
- 고정 5문제

### 예정
- GPT가 코드 복잡도를 판단해서 2~5문제 자율 결정
- 최소 2문제 보장

```
코드 복잡도 LOW  → 2~3문제 (ch01, ch02 수준)
코드 복잡도 MID  → 3~4문제
코드 복잡도 HIGH → 5문제 (디자인 패턴, 자료구조 등)
```

```
Dog 클래스는 Animal을 [ ________ ] 한다.

GPT 판단: ✅ 정답 (extends, 상속, 확장 등 의미상 맞으면 정답)
```

## 7. Firebase 데이터 구조

```
Firestore
├── metaphors/                        # 비유 라이브러리
│   └── {repoName}___{filePath}
│       ├── content: string           # 비유 텍스트
│       ├── functionalAnalysis: string
│       ├── likes: number
│       ├── dislikes: number
│       ├── voters: { uid: 'liked' | 'disliked' }
│       └── updatedAt: timestamp
│
├── learningHistory/                  # 채팅 기록 (예정)
│   └── {uid}_{repoName}_{filePath}
│       ├── messages: array
│       ├── score: number
│       └── completedAt: timestamp
│
└── users/                            # 기존
    └── {uid}
        ├── role: 'student' | 'admin'
        └── groupId: string
```
