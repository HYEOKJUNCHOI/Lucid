# Lucid 코어 시스템 설계

---

## 한 줄 정의
**"어떤 교육 자료든 넣으면, AI가 이해시켜주는 범용 학습 엔진"**

> 최종 목표는 범용 교육 엔진이며, 그 첫 번째 적용 사례로 국비 코딩 교육(레거시)을 선택했다.
> 코딩 교육에 특화된 것이 아니라, 코딩 교육으로 시스템을 검증하고 확장하는 구조.

---

## 코어 아키텍처

```
┌─────────────────────────────────────────────────┐
│                 INPUT (자료 소스)                  │
│                                                   │
│   GitHub API ──→ 코드, md, pdf, docx, xlsx, hwpx  │
│                                                   │
│   강사가 할 일: git push (끝)                      │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│              PARSER (자료 파싱 레이어)              │
│                                                   │
│   .java/.py/.js  → 텍스트 그대로                   │
│   .md/.txt       → 텍스트 그대로                   │
│   .pdf           → pdf-parse (무료)               │
│   .docx          → mammoth.js (무료)              │
│   .xlsx          → sheetjs (무료)                 │
│   .hwpx          → XML 파싱 (내장)                │
│   .hwp           → hwp.js (무료)                  │
│                                                   │
│   출력: 순수 텍스트 + 메타데이터(파일명, 챕터, 순서) │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│               AI ENGINE (핵심)                    │
│                                                   │
│   입력: 파싱된 텍스트                              │
│   출력:                                           │
│     1. 메타포/비유 생성 — "이 개념을 쉽게 설명하면"  │
│     2. AI 채팅 — "이 자료에 대해 질문하세요"        │
│     3. 퀴즈 자동 생성 — "이해도 확인 문제"          │
│     4. 요약 — "이 자료의 핵심 3가지"               │
│     5. 해석 — "이 코드/내용이 하는 일"             │
│                                                   │
│   모델: GPT-4.1-nano (속도+비용 최적화)            │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│               OUTPUT (학습 경험)                   │
│                                                   │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│   │  루틴     │ │ 챕터학습  │ │ 문제지옥  │        │
│   │ (복습)    │ │ (탐색)   │ │ (실력측정)│        │
│   └──────────┘ └──────────┘ └──────────┘        │
│                                                   │
│   공통 기능:                                      │
│     - 메타포 (비유 해석)                           │
│     - AI 채팅 (질문 답변)                          │
│     - 퀴즈 (이해도 확인)                           │
│     - 사전 (용어 팝업)                             │
│     - XP/streak (동기부여)                         │
│     - 하트/LP (피드백/랭킹)                        │
└─────────────────────────────────────────────────┘
```

## 왜 "범용"인가

### 강사가 GitHub에 올리는 것만 바꾸면 된다

| 수업 종류 | GitHub에 올리는 자료 | Lucid가 하는 일 |
|----------|---------------------|----------------|
| Java 기초 | Main.java | 코드 해석 + 비유 + 퀴즈 |
| Python AI | notebook.ipynb | 개념 설명 + 퀴즈 |
| 웹개발 | index.html + style.css | 구조 해석 + 퀴즈 |
| SQL | query.sql | 쿼리 해석 + 퀴즈 |
| 이론 수업 | 개념정리.md | 요약 + 퀴즈 |
| 자격증 | 기출문제.pdf | 문제 분석 + 유사 퀴즈 생성 |
| 실무 교육 | 매뉴얼.docx | 핵심 요약 + 확인 퀴즈 |
| 한국어 교안 | 교안.hwpx | 내용 파싱 + 퀴즈 |

**엔진은 하나. 콘텐츠만 교체.**

## 학습 모드별 역할

| 모드 | 목적 | 사용 도구 | 핵심 가치 |
|------|------|----------|----------|
| 루틴 | 매일 복습 | 퀴즈 + 하트 | 습관 형성 (2~3분) |
| 챕터별 학습 | 자료 탐색 + 이해 | 코드뷰 + AI채팅 + 메타포 | 깊은 이해 |
| 자유학습 | 자기주도 탐색 | AI 코드 생성 + 채팅 | 호기심 해결 |
| 문제지옥 | 실력 측정 | 퀴즈 + LP 랭킹 | 실력 객관화 |

## 동기부여 시스템

### XP (전체 레벨)
- Lv.1~99 / 성실도 척도
- 루틴, 학습, 문제지옥 모두에서 획득

### LP (문제지옥 전용)
- 과목별 티어 (아이언~다이아)
- 실력 척도 / 시즌 리셋 (월간)

### streak (연속 학습)
- 매일 루틴 완료 시 +1
- 습관 형성의 핵심 지표

### 하트 (루틴 전용)
- 시각적 피드백 (탈락 없음)
- 틀리면 라이팅 + shake 연출

---

## 두 축 시스템 — 성실성 vs 실력

### XP + streak = 성실성의 척도
- 루틴만 꾸준히 해도 XP가 쌓이고 streak이 이어진다
- **"이 학생은 학습을 지속하고 있다"**는 신호 — 그 자체로 나쁘지 않음
- 매일 2~3분 루틴 = 습관 형성 = 성실성 증명
- 강사 입장: streak이 이어지는 학생은 최소한 관심을 놓지 않은 것

### LP + 티어 = 실력의 척도
- 문제지옥에서만 측정되는 별도 등급 시스템
- 아이언 → 브론즈 → 실버 → 골드 → 플래티넘 → 다이아
- 같은 개념을 얼마나 깊이 이해하는지로 티어 결정
- 골드 = 수업 + 복습 하면 도달 (수료 기준선)
- 플래티넘~ = 진짜 실력 (강사 커버 범위 밖)

### 두 축이 교차하면 학생 상태가 보인다

| | 티어 높음 | 티어 낮음 |
|--|:---:|:---:|
| **streak 높음** | 놔두면 됨 | 열심히 하는데 이해 못함 → 면담 |
| **streak 낮음** | 머리 좋은데 안함 → 동기부여 | 위험 학생 |

- 강사 대시보드에서 학생별 streak + 티어 교차 확인
- **강사/관리자가 학생을 판단할 수 있는 기준** — 감이 아닌 데이터로 면담, 동기부여, 위험 감지

## 핵심 차별점

1. **강사는 GitHub에 push만 하면 끝** — 별도 관리자 페이지에서 자료 업로드 불필요
2. **어떤 파일이든 소화** — 코드, 문서, PDF, 한글, 엑셀 전부
3. **AI가 자동으로 학습 콘텐츠 생성** — 메타포, 퀴즈, 해석을 사람이 만들지 않음
4. **특정 과목에 종속되지 않는 범용 엔진** — Java든 Python이든 이론이든
5. **학생 동기부여 시스템 내장** — XP, streak, LP, 하트
6. **강사 대시보드** — 학생 상태 한눈에 파악, 면담 기준 자동화
