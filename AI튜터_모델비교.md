### gpt-4.1-nano (프롬프트 개선 후)

```
싱글톤 패턴으로 카페 인스턴스 하나만 만들어서 관리하는 코드야.

`카페` 클래스는 하나의 인스턴스만 생성되게 하려고
`getInstance()` 메서드로 통제하고 있어.

이 인스턴스는 여러 곳에서 호출해도 항상 같은 객체를 반환하지.

`order()` 메서드로 메뉴 주문을 받고, 주문 수를 세는 것도 들어 있어.
```

---

### gpt-4.1-mini

```
## 한 줄 요약 🚀
싱글톤 패턴으로 하나뿐인 카페 객체 만들어서 주문 관리하는 코드야.

## 설명
- `카페` 클래스는 딱 한 개 인스턴스만 생성돼.
- `getInstance()` 가 그 인스턴스 가져오고, 없으면 새로 만들어줘.
- `order()` 메서드는 주문할 때마다 주문 수를 1씩 올리고 출력해.
- `main` 에서 `카운터1` 과 `카운터2` 가 같은 객체임을 확인해.

## 비유 👍
카페가 한 곳뿐인데,
여러 카운터에서 주문 받는 느낌이야.
그래서 주문 수가 모두 공유됨!
```

---

### gpt-4o (1차)

```
싱글톤 패턴을 쓰는 코드야.

## 핵심 설명
- 싱글톤 패턴: 객체를 한 번만 생성해서 어디서든 같은 객체를 쓰는 방법이야.

## 코드 흐름
1. `카페` 클래스는 `instance` 라는 정적 변수를 가지고 있어.
2. `getInstance()` 메서드를 호출하면 `카페` 객체가 없을 때만 새로 만들어.
3. 이미 있으면 그걸 그냥 돌려줘.
4. 그래서 `카페` 객체는 하나만 존재해.

## 비유
마치 스타벅스 강남점이 하나라서
어디서 주문해도 같은 매장에 주문하는 것과 같아.
```

### gpt-4o (2차 — 같은 프롬프트)

```
싱글톤 패턴으로 카페 객체 하나만 만들고, 주문을 관리하는 코드야.

## 핵심
- 싱글톤 패턴: 객체를 하나만 생성.
- `order` 메서드로 주문 수 관리.
- `getInstance` : 이미 있는 객체 반환, 없으면 새로 만듦.

비유하자면, 스타벅스 강남점이 하나뿐이라, 여러 카운터가
있어도 같은 가게에서 주문을 처리하는 느낌이야.
```

> ⚠️ 1차보다 구조 단순 — 응답 일관성이 낮음
