## 2. 루틴 XP

### 기본 구성 (하루 최대 250)

| 항목 | XP |
|------|----|
| 출석 체크 | 50 |
| 루틴 1세트 — OX 문제 | 50 |
| 루틴 2세트 — 객관식 | 50 |
| 루틴 3세트 — 빈칸채우기 | 100 |
| **합계** | **250** |

### 반복 드랍률

| 회차 | 지급률 |
|------|--------|
| 1회차 | 100% |
| 2회차 | 50% |
| 3회차 이후 | 최소치 (5~10 XP 고정) |

- 반복 자체는 허용. XP 파밍만 방지.
- 재루틴 클리어 시 "저번보다 빨리 풀었어요" 피드백 제공 (응답시간 비교)

### 루틴 진도 우선순위

```
1순위: 오늘 강사가 GitHub에 올린 코드
2순위: 강사가 등록한 예상 커리큘럼 순서
3순위: 문제지옥 Java 기본 트랙 (fallback)
```

- 루틴 클리어 시 해당 챕터 복습 완료 처리
- 모든 진도 소진 시 자유 예습 공간으로 자동 전환

## 3. 자유 예습 공간 XP

- 학생이 주제 직접 입력 → AI 예시 코드 생성 → 루틴 3세트 동일 구성 제공
- XP: 소량 고정 지급 (정확한 수치는 추후 조정)
- 과목별 티어에 반영 안 됨
- 학습 히스토리로 기록됨

## 4. 문제지옥 XP

### 난이도별 단가 및 뱃지

| 난이도 | 뱃지 | XP |
|--------|------|----|
| 쉬움 | 🟢 | 5 |
| 보통 | 🟡 | 10 |
| 어려움 | 🔴 | 25 |

- 문제 카드에 뱃지 항상 노출
- 학생이 문제 보기 전에 난이도와 XP를 인지할 수 있어야 함

### 세션 구성 (약 30분 / 28~30문제)

| 구간 | 난이도 | 문제 수 | XP |
|------|--------|---------|-----|
| 1~10문제 | 🟢 쉬움 (예열) | 15 | 75 |
| 11~20문제 | 🟡 보통 (본게임) | 10 | 100 |
| 21~30문제 | 🔴 어려움 (마무리) | 3 | 75 |
| **합계** | | **28문제** | **250** |

### 중도 하차 처리

- 푼 문제 수만큼 분할 지급
- 진행률 90% 이상 시 "N문제만 더하면 달성!" 표시 → 이탈 방지 유도

## 5. 배치고사 (건너뛰기)

> 전공자 또는 특정 레벨을 이미 아는 학생이 레벨 점프를 원할 때 사용

### 배치고사 구성

| 난이도 | 뱃지 | 문제 수 | 점수 |
|--------|------|---------|------|
| 쉬움 | 🟢 | 2문제 | 5점 × 2 = 10점 |
| 보통 | 🟡 | 6문제 | 10점 × 6 = 60점 |
| 어려움 | 🔴 | 2문제 | 25점 × 2 = 50점 |
| **합계** | | **10문제** | **만점 120점** |

### 레벨 점프 기준 (가중치 합산)

| 점수 | 정답률 | 레벨 점프 |
|------|--------|-----------|
| 108~120점 | 90% 이상 | +5 레벨 |
| 96~107점 | 80% 이상 | +4 레벨 |
| 84~95점 | 70% 이상 | +3 레벨 |
| 72~83점 | 60% 이상 | +2 레벨 |
| 60~71점 | 50% 이상 | +1 레벨 |
| 60점 미만 | 50% 미만 | 실패, 재도전 가능 |

- 어려움 2문제 모두 맞으면 50점 → 쉬움/보통 일부 틀려도 커버 가능 (실력자 우대)
- 배치고사 XP는 별도 지급 없음 (레벨 점프가 보상)

## 6. 적응형 난이도 (Adaptive Difficulty)

### 난이도 조정 기준 (전날 정답률 기준)

| 정답률 | 조치 |
|--------|------|
| 80% 이상 | 난이도 1단계 상승 |
| 50~80% | 유지 |
| 50% 미만 | 난이도 1단계 하강 |

- GPT 프롬프트 파라미터로 제어 (Firebase 정답률 → 프롬프트 난이도 파라미터 주입)
- 티어 오를수록 쉬움 기준도 올라감 (항상 예열→본게임→마무리 리듬 유지)

### 티어 강등 조건

- 7일 연속 정답률 50% 미만 → 해당 과목 티어 1단계 하강
- Firestore `tier` 필드 값 -1 처리

## 8. 루시드 전체 레벨 (스테이터스 카드 표기)

> 스테이터스 카드에 표기되는 유일한 레벨. 이름 아래 티어 표기 없음.

- 최고 레벨: **Lv.99** (만렙)
- 레벨당 필요 XP: **디아블로 곡선** (초반 빠름 → 후반 급격히 느려짐)
- 공식: `레벨당 XP = 50 + 2n + 0.15n²` (n = 현재 레벨)
- 만렙까지 총 필요 XP: **약 45,000 XP**

### 진행 기준 (평일 기준 출석)

| 시점 | 루틴+문제지옥 (500 XP/일) | 루틴만 (250 XP/일) |
|------|--------------------------|-------------------|
| 1주 (5일) | **Lv.14** | Lv.10 |
| 1개월 (22일) | **Lv.30** | Lv.22 |
| 2개월 (44일) | **Lv.65** | Lv.48 |
| 3개월 (66일) | **Lv.75** | Lv.65 |
| **90일 개근** | **Lv.99 (만렙)** | **Lv.80** |

### 설계 의도

- **99는 가능하지만 조건부** — 루틴 + 문제지옥 + 90일 개근 모두 충족해야 달성
- **루틴만 90일 개근 → Lv.80** — 성실함은 인정, 만렙은 불가
- **이탈하면 절대 99 못 찍음** — 개근 자체가 가장 큰 보상 조건
- 개근이 학원/멘토에게 가장 중요한 가치 (이탈 방지 = 수익 유지)

### 스테이터스 카드 구성

```
[ 레벨 아이콘 ]  이름
                (티어 표기 없음)
Lv.N                        N XP
[===============----------] 진행 바
```

- 이름 아래 티어 없음 — 레벨 숫자만으로 표현
- 레벨은 시간과 성실의 총합

## 9. 과목별 티어 (문제지옥 카드에서만 표기)

> 전체 레벨과 별개. 문제지옥 과목 카드 하단에서만 노출.

### 티어 구간 및 디자인

| 티어 | 색상 | 글로우 |
|------|------|--------|
| 아이언 | 회색 | 없음 |
| 브론즈 | `#CD7F32` 주황 | 없음 |
| 실버 | `#C0C0C0` | 없음 |
| 골드 | `#FFD700` | 골드 글로우 |
| 플래티넘 | `#E5E4E2` | 약한 글로우 |
| 다이아 | `#FFFFFF` bold | 시안 글로우 |

### 티어별 CSS

```css
.tier-bronze  { color: #CD7F32; }
.tier-silver  { color: #C0C0C0; }
.tier-gold    { color: #FFD700; border: 1px solid #FFD700; box-shadow: 0 0 8px rgba(255,215,0,0.5); }
.tier-diamond { color: #FFFFFF; font-weight: 700; border: 1px solid #67E8F9; box-shadow: 0 0 12px rgba(103,232,249,0.6); }
```

### 카드 하단 진행 바

```
브론즈 0% → 브론즈 100% → 실버 → 골드 → 다이아
████░░░░░░  40%
```

### 과목 목록

- Java / React / Python / Git / 정보처리기사 각자 독립
- Java 다이아인데 Python 실버 가능

## 10. XP → 재화 → 쿠폰 환산

```
1 XP = 1원 환산 기준 (조정 가능)

루틴만 매일 250 × 14일 = 3,500원 → 2,000원 커피 쿠폰
루틴만 매일 250 × 28일 = 7,000원 → 4,000원 아이스아메리카노 쿠폰
```

- 쿠폰 지급은 멘토/학원 측에서 수동 or 자동 발급 (v1은 UI만)
- 지급 타이밍 = 멘토가 학생에게 자연스럽게 연락하는 접점

## 11. Firestore 데이터 구조 (참고)

```
user/{uid}
  - dailyXP: number              // 오늘 획득 XP (자정 리셋)
  - totalXP: number              // 누적 XP
  - level: number                // 루시드 전체 레벨 (1~99)
  - streak: number               // 연속 출석 일수
  - lastStudiedAt: timestamp

subjectTier/{uid}/{subject}
  - tier: string                 // 과목별 티어
  - tierProgress: number         // 티어 내 진행률 0~100%
  - correctRate: number          // 최근 정답률 (적응형 난이도 기준)
  - consecutiveLowRate: number   // 연속 저조 일수 (강등 카운터)
  - currentLevel: number         // 현재 레벨 (배치고사 점프 반영)
```
