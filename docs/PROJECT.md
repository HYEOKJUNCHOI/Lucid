# PROJECT.md — Lucid

---

## ★ 1. 프로젝트 개요

- **프로젝트명:** Lucid
- **한줄 요약:** 국비 부트캠프 학생이 오늘 배운 코드를 불러오면, AI가 내 언어로 설명하고 내 레벨을 알려준다. 학원은 그 데이터로 학생을 관리한다.
- **핵심 전제:** 사람은 새로운 정보를 머릿속에 있는 것과 연결할 때 비로소 이해한다. Lucid는 오늘 배운 코드를 학생이 이미 아는 언어로 번역해서 그 연결을 만든다.
- **버전:** v1.0.0 (공모전 제출용)

---

## ★ 2. 설계 철학

> **Scott Adams:** "학생이 혼자 GPT를 켠 순간이 있다. 그 순간이 Lucid가 있어야 할 자리다."
>
> **Nir Eyal:** "학생이 앱을 여는 건 알림 때문이 아니다. '나 지금 어디 있지?' 라는 불안 때문이다. 그 불안을 받아주는 시스템을 만들어라."

- **핵심 가치:**
  - 학생에게는 거울 — 내 수준이 보인다
  - 학원에게는 레이더 — 학생 상태가 보인다
  - AI가 가르치는 게 아니라, 학생이 스스로 발견하게 한다
  - 말(자기보고)을 믿지 않는다. 행동 패턴을 본다.

## ★ 3. 타겟 유저

| 구분 | 설명 |
|------|------|
| 주요 사용자 | 국비 지원 IT 부트캠프 수강생 / 강사 / 멘토(학원 운영자) |
| IT 숙련도 | 수강생: 비전공자 입문 / 강사: 중급 이상 / 멘토: 비개발자 |
| 사용 환경 | PC 웹 브라우저 + 모바일 반응형 |
| 핵심 불편 | **수강생** — 뭘 해야 할지 모름. 강사한테 창피해서 질문 못 함. 내 레벨을 모름. |
| | **강사** — 누가 이해했는지 모름. 평균에 맞춰 가르칠 수밖에 없음. |
| | **멘토** — 누가 이탈하려는지 모름. 영업이 본업이라 수강생 관리가 후순위. |

> 핵심 통찰: 학생도 모르고, 강사도 모르고, 멘토도 모른다. 서로의 상태를 아무도 모른다. 그러다 어느 날 학생이 사라진다.

## ★ 4. 핵심 기능 목록

### 로그인 화면
- [ ] Google 로그인 (Firebase Auth)
- [ ] 관리자 계정 감지 → 관리자 토큰 발급 → 관리자 페이지 접근 허용
- [ ] 일반 학생 → 학생 화면으로 이동
- [ ] UI: Claude 화면 구도 참고 (다크, 심플, 중앙 로그인 박스)

### 학생 화면
- [ ] 강사 선택 드롭다운 → 레포 주소 자동 연결
- [ ] 진입 모드 선택: 챕터 모드 / 날짜 모드
  - 챕터 모드: 개념 단위 선택 (배열, for문, 클래스 등)
  - 날짜 모드: 달력에서 날짜 선택 → 그날 커밋 불러오기
- [ ] GitHub API로 강사 레포 코드 추출
- [ ] 개념 선택 UI (다크모드, 형광 컬러)
- [ ] PC: 왼쪽 코드 보기 / 오른쪽 GPT 채팅창
- [ ] 모바일: 코드 탭 / 채팅 탭 전환
- [ ] 게임 메타포 + 생활 메타포 동시 제공
- [ ] 메타포 좋아요 버튼 → Firebase 저장 + 라이브러리 likeCount +1
- [ ] 문제 출제 (4지선다) → 틀리면 다른 메타포로 재설명
- [ ] 학습 종료 시 레벨 결과 화면 (클래시오브클랜 스타일 별/트로피)

### 동기부여 시스템 (학생 화면)
- [ ] 나 vs 어제의 나 — 개인 성장 그래프
- [ ] 오늘 함께 공부 중인 학생 N명 표시
- [ ] "오늘 X명이 Y챕터를 깼어요" — 익명 사회적 연결
- [ ] 전체 랭킹 없음 (소극적 학생 배려)

### 관리자 화면
- [ ] 학생 정보 입력 (테스트용)
- [ ] 강사 정보 + GitHub 레포 등록
- [ ] 전체 학생 레벨 대시보드
- [ ] 이탈 위험 신호 (🔴 3일 미접속 / 🟡 레벨 제자리)
- [ ] 이탈 감지 알림 UI (실제 발송은 v2)

- [ ] 우수자 쿠폰/리워드 UI (실제 발급은 v2)
- [ ] 메타포 라이브러리 관리자 페이지

## ★ 5. 사용자 여정 (User Flow)

```
[로그인]
접속
→ Google 로그인
→ 관리자 계정? → 관리자 토큰 발급 → 관리자 화면
→ 일반 학생?   → 학생 화면

[학생]
→ 강사 선택 (레포 자동 연결)
→ 모드 선택: 챕터 / 날짜
→ 개념 선택
→ 로딩 → 페이지 전환
   PC: 왼쪽 코드 원문 / 오른쪽 GPT 채팅
   모바일: 코드 탭 / 채팅 탭
   - 게임 메타포 설명
   - 생활 메타포 설명
   - 좋아요 버튼
   - 문제 출제 → 정답/오답 → 다른 메타포 or 다음 개념
→ 학습 종료
→ 레벨 결과 화면 (별/트로피)
→ Firebase 저장 (레벨 + 좋아요 목록)

[관리자]
→ 학생/강사 정보 입력
→ 전체 대시보드
→ 🔴🟡 신호 학생 확인
→ 주간 TOP 3 확인 → 우수자 선발
→ 좌석 배치도 확인 / 수정
```

## ★ 7. 아키텍처

```
[로그인]
  └─ Firebase Auth → role 판단 → 라우팅

[학생 브라우저]
  │
  ├─ GitHub API → 강사 레포 커밋 → Java 파일 추출
  │
  ├─ GPT API
  │    ├─ Agent 1: 코드 분석 → 개념 추출
  │    ├─ Agent 2: 메타포 생성 (게임 + 생활)
  │    └─ Agent 3: 레벨 진단 (개념별 5문제: 개념→규칙→제약→적용→심화, 전부 맞으면 클리어, 레벨 = 클리어 개념 수)
  │
  └─ Firebase
       ├─ user/{uid}: role, 레벨, 좋아요 목록, 마지막 접속
       └─ metaphors/{id}: 개념별 메타포 라이브러리 (likeCount)

[관리자 브라우저]
  └─ Firebase → 학생 데이터 조회 → 대시보드 + 좌석 배치 + TOP3
```

## ★ 8. 데이터 모델

```
user/{uid}
  - role: "student" | "admin"    // 역할 구분
  - currentLevel: number          // 현재 레벨
  - likedMetaphors: string[]      // 좋아요 누른 메타포 id 목록
  - lastStudiedAt: timestamp      // 마지막 학습일

teachers/{id}
  - name: "김준일"
  - repoUrl: "https://github.com/code1218/korit_9_gov_java"

metaphors/{id}
  - concept: string               // "배열", "for문", "클래스"
  - type: "game" | "life"         // 메타포 종류
  - content: string               // 설명 블록 전문
  - likeCount: number             // 좋아요 수
  - createdAt: timestamp
```
